# 词条系统核心规则（修正版）

## 一、词条合成规则（已修正）

### 1. 合成消耗机制
- **合成成功后：两个材料词条永久消失，生成新词条**
- **合成失败后：两个材料词条永久消失，无新词条生成**
- 无论成功与否，材料词条都不会保留！

### 2. 成功率与风险
| 合成等级 | 成功率 | 失败惩罚 |
|---------|--------|---------|
| F→E | 80% | 损失两个F级词条 |
| E→D | 70% | 损失两个E级词条 |
| D→C | 60% | 损失两个D级词条 |
| C→B | 50% | 损失两个C级词条 |
| B→A | 40% | 损失两个B级词条 |
| A→S | 30% | 损失两个A级词条 |
| S→SS | 20% | 损失两个S级词条 |
| SS→SSS | 10% | 损失两个SS级词条 |

### 3. 萧云的关键策略
- **前期（F-E级）**：大量收集F级词条，批量合成E级，利用80%高成功率堆数量
- **中期（D-B级）**：谨慎合成，确保有足够备份材料，避免一朝回到解放前
- **后期（A-SSS级）**：使用"保底机制"（需要特殊道具或条件），确保高价值材料不会白白浪费

## 三、写作规范（防止词条爆炸）

### 1. 词条获取频率控制
- **每3章最多获得1个E级以上词条**（防止高级词条泛滥）
- **每10章最多进行1次D级以上合成**（高风险操作要慎重）
- **F级词条不限数量**，但要有选择地携带（最多同时携带5个F级）

### 2. 战斗描写规范
- 每次战斗前，明确列出萧云当前携带的所有词条
- 战斗中，每次使用词条能力都要明确点名
- 战斗结束后，更新词条状态（消耗、损坏、新获得等）

### 3. 合成场景描写规范（重点！）
- **合成前**：明确写出"两个材料词条即将消失"
- **合成中**：描写过程紧张感，强调"一旦失败，两个词条都将消失"
- **合成成功后**：明确写出"两个材料词条已消失，新词条诞生"
- **合成失败后**：明确写出"两个材料词条已消失，一无所获"（重要！不是保留！）

### 4. 示例对比（错误 vs 正确）

❌ **错误写法**（会导致词条爆炸）：
> 萧云将【坚不可摧】和【致命暴击】合成【破军之力】，现在他有3个词条了。
> （错误！合成后材料词条消失，应该只剩【破军之力】1个！）

✅ **正确写法**（词条管理清晰）：
> 萧云深吸一口气，将【坚不可摧(F)】和【致命暴击(F)】放入合成栏。
> 【系统提示：合成成功率为80%，一旦失败，两个材料词条将永久消失，是否确认？】
> "确认！"
> 【合成中……10%……50%……90%……】
> 【合成成功！！！】
> 【坚不可摧(F)】已消失！
> 【致命暴击(F)】已消失！
> 获得新词条：【破军之力(E)】！
> 当前携带词条：【破军之力(E)】×1

## 四、总结

### 核心原则：词条是消耗品，不是收藏品！

1. **合成必然消耗材料词条**（无论成功与否）
2. **同时携带的词条数量有限**（需要取舍和策略）
3. **高级词条获取困难**（不能泛滥，要有稀缺性）
4. **每场战斗都要明确词条状态**（防止逻辑混乱）

只有严格遵守这些规则，才能避免出现"词条爆炸""状态混乱"等问题，让系统设定严谨可信，让读者阅读流畅清晰。
