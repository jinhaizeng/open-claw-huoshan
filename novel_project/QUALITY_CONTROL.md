# 小说质量管控体系

> 确保每章连贯、状态无冲突、剧情丝滑

---

## 一、核心原则

### 1. 状态连续性原则
- **每章开始必须从上一章结束状态严格承接**
- **任何状态变化必须在章节中明确记录**
- **禁止状态跳跃或模糊处理**

### 2. 信息一致性原则
- **同一信息在不同章节必须一致**
- **人物、地点、物品属性不能矛盾**
- **时间线必须清晰连贯**

### 3. 剧情丝滑原则
- **每章必须有明确起承转合**
- **结尾必须留钩子（悬念/期待）**
- **爽点和剧情推进要平衡**

## 二、状态追踪机制

### 主角状态表（每章必填）

| 属性 | 类型 | 说明 | 示例 |
|------|------|------|------|
| **当前词条** | 列表 | 携带的所有词条 | 【破军之力(E)】×1 |
| **已消耗词条** | 列表 | 已消失/合成的词条 | 【坚不可摧(F)】❌ |
| **当前寿命** | 数字 | 剩余寿命年份 | 98年 |
| **当前境界** | 文字 | 修为等级 | 炼体境三重 |
| **当前位置** | 文字 | 所在地点 | 青云宗后山 |
| **关键物品** | 列表 | 重要道具 | 无 |
| **本章目标** | 文字 | 章节任务 | 收集更多词条 |

### 状态变化记录模板

```
【第X章 - 状态变化记录】

▶ 章节开始状态：
- 词条：【xxx】×N
- 寿命：XX年
- 境界：xxx
- 位置：xxx

▶ 本章变化：
1. 【事件】击败XX → 【变化】获得词条【xxx】
2. 【事件】合成词条 → 【变化】【xxx】+【xxx】=【xxx】，原词条消失
3. 【事件】突破境界 → 【变化】炼体X重 → 炼体Y重

▶ 章节结束状态：
- 词条：【xxx】×N（明确列出所有）
- 寿命：XX年（精确数字）
- 境界：xxx（明确等级）
- 位置：xxx（明确地点）

⚠️ 关键提醒：
- 下一章必须从此状态严格承接
- 任何状态变化必须在下一章明确记录
```

## 三、章节质量检查清单

### 每章开始检查（写作前）
- [ ] 已查看上一章结束状态
- [ ] 已确认主角当前词条、寿命、境界
- [ ] 已明确本章目标和关键事件
- [ ] 已设计本章爽点和钩子

### 每章结束检查（写作后）
- [ ] 已填写状态变化记录
- [ ] 已明确列出当前所有词条
- [ ] 已确认寿命和境界变化
- [ ] 已设计结尾钩子（悬念/期待）
- [ ] 已检查与上一章无矛盾

### 每3章深度检查
- [ ] 回顾主角成长曲线是否合理
- [ ] 检查是否有遗漏的状态变化
- [ ] 确认时间线是否连贯
- [ ] 评估爽点密度是否足够

## 四、常见问题处理

### Q1：发现前后章节状态矛盾怎么办？
**处理流程：**
1. 立即停止写作
2. 回溯查找矛盾源头
3. 选择影响最小的修正方案
4. 修正所有相关章节
5. 更新状态追踪表
6. 继续写作

### Q2：不小心让主角状态太强/太弱怎么办？
**处理流程：**
1. 评估当前状态与大纲的偏差
2. 设计合理的剧情事件调整状态
3. 通过战斗/消耗/意外等自然调整
4. 避免突兀的机械降神

### Q3：发现剧情和设定有冲突怎么办？
**优先级：**
1. 人物设定 > 世界观设定 > 剧情需要
2. 已写内容 > 未写大纲
3. 选择影响最小的修改方案
