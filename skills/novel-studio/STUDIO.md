# 玄幻小说编辑工作室 (Novel Studio)

> 一个模拟小型出版公司的 AI Agent 协作系统，专注于玄幻/修仙类网文的全流程编辑。

                   ▼

       │          │          │

## 数据层

所有项目数据存放在 `workspace/novel_project/` 下：

```
novel_project/
├── settings.json     # 世界观设定
├── outline.md        # 全书大纲
├── chapters/         # 正文章节
├── lore_db/          # 设定数据库
│   ├── characters.json
│   ├── items.json
│   ├── locations.json
│   ├── timeline.json
│   ├── factions.json
│   └── unconfirmed.json
├── audits/           # 审查报告
├── style_guide/      # 风格指南
│   ├── samples.md
│   ├── vocabulary.md
│   └── taboos.md
└── revisions/        # 修改版本
```

## 标准工作流 (SOP)

### 流程 A: 新项目启动
1. User 提供创意灵感 → 主编接收
2. 主编 → librarian 初始化项目目录
3. 主编 → 生成 settings.json + outline.md
4. librarian → 从 settings.json 提取初始设定入库
5. User 审批

### 流程 B: 章节编辑（核心）
1. User 提交初稿 / 主编指派写作
2. librarian 扫描初稿，提取新实体 → unconfirmed.json
3. 三编并行审稿 → audits/
4. 主编汇总终审 → 通过/修改/重写
5. 如需修改 → copy-editor 执行（最多3轮）
6. 定稿 → librarian 确认实体入库

### 流程 C: 人工干预
- 三编冲突 / 重大转折 / 3轮未解决 → 呼叫用户

## Agent 通信
所有 Agent 通过文件系统通信（写文件 → 读文件），不直接调用彼此。

## 技术约束
- **长文本**: 每次只加载当前章节 + 前3章摘要 + lore_db 相关条目
- **幻觉控制**: 战力/功法/法宝必须先查 lore_db 确认
- **风格统一**: style_guide/ 提供 few-shot + 词汇表 + 禁忌
- **审稿冲突**: 设定一致性 > 剧情合理性 > 文字优美度
